# OutEye 研究助手桌宠系统 — 情绪化陪伴体验设计规范

> 版本: v3.0-emotional  
> 状态: 设计阶段  
> 角色: 小外（XiaoWai）· Maltese 研究守护精灵  
> 视觉方向: 手绘感 × 轻学术 × 时间氛围 × 科研共情  
> 核心原则: **情绪增强，但不娱乐化**

---

## 一、角色人格（v3 深化）

### 1.1 一句话描述

一只安静趴在你科研手账角落的小犬。它知道你什么时候在熬，什么时候在等，什么时候终于完成了。

### 1.2 人格层次

```
表层（用户可见）:  安静、温柔、偶尔小动作
中层（情绪响应）:  读懂科研节奏，在合适的时候给出微弱的共情信号
底层（设计哲学）:  不是宠物，不是助手，是「研究过程的见证者与守护者」
```

### 1.3 小外「懂」的时刻

| 场景 | 小外的理解 | 回应 |
|---|---|---|
| 你盯着同一篇文献 5 分钟没动 | "你在认真读" | 安静趴着，呼吸变慢 |
| 进度条卡了 30 秒不动 | "卡住了吗？" | 爪子轻轻伸向进度条方向 |
| 深夜还在工作 | "你也在熬夜啊" | 眼睛变半闭，身体靠你更近 5px |
| 你连续快速操作 | "在冲刺了" | 不打哈欠、不做慢动作，保持专注陪伴 |
| 你终于完成任务 | "做到了" | 站起来，前爪贴额头（轻敬礼） |
| 你刚打开页面 | "早上好" | 根据时间给不同的晨间动作 |

## 二、视觉层：去工业化（v3 重大升级）

### 2.1 轮廓线 — 手绘感线条系统

**核心**: 线条不再像 SVG `<path>`，而像蘸水钢笔在纸上轻轻划过。

**实现原则**:
- 主线（身体轮廓）: 2.5px 粗细，带极其微弱的自然抖动（±0.3px 不规则偏移）
- 内线（五官、围巾褶皱）: 1.2px 粗细，比主线更轻
- 线条末端: `stroke-linecap: round`，模拟钢笔起笔和收笔的微圆头
- 转角处: 略微加粗 0.3px（模拟手绘时转角墨水聚集），但极其克制

**SVG 实现方案**:
```
方案 A（推荐）: SVG feTurbulence + feDisplacementMap
  - 对轮廓线 path 施加极低强度的 turbulence 滤镜
  - baseFrequency: 0.05, scale: 0.3
  - 效果: 线条有肉眼可见但不过分的自然不规则感

方案 B（备选）: 双路径叠加
  - 主路径 + 偏移 0.3px 的半透明副路径
  - 模拟铅笔/钢笔的轻微溢墨效果
```

**严禁**: 
- ✗ 粗糙的素描抖动（会变成儿童画风）
- ✗ Photoshop 滤镜效果（不适用于 SVG 实时渲染）
- ✗ 虚线或点线（破坏"陪伴"的稳定感）

### 2.2 品种特征 — 轻盈长毛感

Maltese 标志性的长毛特征，用极简线条表达：

```
耳朵边缘飘逸毛:
  每只耳朵外侧 2-3 根极细曲线 (1px, opacity 0.4)
  长度不超过耳朵本体的 1/3
  微微弯曲，像被静电轻轻吸起
  动画: 每 5-10 秒微微飘动一次（像呼吸气流带动）

脸颊绒毛:
  脸颊两侧各 1-2 根极短细线 (0.8px, opacity 0.3)
  向上微翘，长度不超过头部宽度的 1/10
  静止不动（不参与动画，保持安静）

爪间绒毛:
  前爪之间 1 根短弧线 (0.8px, opacity 0.3)
  仅在伸懒腰时可见爪子抬起瞬间
```

**克制原则**: 每个部位最多 3 根细毛线，整体飘逸毛线不超过 8 根。不是为了"好看"，而是为了"触感"——让用户感觉小外是柔软的。

### 2.3 材质细节层

#### 围巾针织纹理

```
围巾底色: 品牌蓝 #4A90A4
针织纹理: 极其微弱的十字纹（SVG pattern）
  - 单元: 2px × 2px 的微十字
  - 颜色: rgba(255,255,255,0.04) — 几乎看不见
  - 效果: 不是装饰，是"凑近看能感觉到织物"的微妙质感
  - 动画: 围巾飘动时纹理跟随 transform
```

#### 莫兰迪低饱和备选色系

```
标准色（默认）: 品牌蓝 #4A90A4
备选色:
  --scarf-dusty-rose:    #C4A8A0   莫兰迪灰粉（适合偏好暖调的用户）
  --scarf-sage:          #A8B5A0   莫兰迪灰绿（适合偏好冷调的用户）
  --scarf-warm-grey:     #B8B0A8   莫兰迪暖灰（最克制的选项）
  --scarf-lavender-grey: #B0A8B8   莫兰迪灰紫（深夜模式推荐）

所有备选色饱和度 < 15%，确保不刺眼、不抢注意力
```

#### 眼镜起雾效果

```
触发: 小外打哈欠后 0.5s
表现: 眼镜下半部分覆盖半透明白色椭圆
  - opacity: 0.15 → 0.05（5秒内自然消散）
  - 像冬天呼出的雾气在眼镜上的极速消散版
意义: 增加"它是一个生物"的微妙暗示，不是装饰
```

## 三、时间氛围系统

### 3.1 设计原则

- 氛围光**仅影响小外周围 180px 半径**，绝对不改变页面其他区域的视觉
- 使用 CSS radial-gradient 实现，不依赖 SVG filter（性能安全）
- 过渡时间: 15 分钟平滑渐变（模拟自然光线变化，不跳变）

### 3.2 时段氛围定义

```
🌅 早晨 (06:00 - 10:00) — 「晨间研究」
  光色: 暖金色, #F5E6C8
  强度: opacity 0.12 → 非常克制
  光型: 从小外右上角斜照（模拟晨光从窗户进来）
  小外状态: 刚醒来，围巾整理好，眼睛清亮
  特殊: 如果用户今天第一次打开 OutEye，小外会轻轻"抬头"3px（欢迎）

☀️ 白天 (10:00 - 17:00) — 「专注研究」
  光色: 无额外光（自然光充足）
  小外状态: 普通趴姿，正常呼吸
  特殊: 这是小外最"透明"的时段——完全融入工作流，不提供额外氛围

🌆 傍晚 (17:00 - 20:00) — 「黄昏研究」
  光色: 暖琥珀, #F0D5B0
  强度: opacity 0.10
  光型: 从下方漫反射（模拟桌灯从桌面反射）
  小外状态: 围巾微微松垮（一天下来），身体略微更靠近桌面
  特殊: 如果用户连续工作 >6h，此时光色略微偏暖（多 0.02 opacity）

🌙 深夜 (20:00 - 06:00) — 「深夜研究」
  光色: 双色 — 暖黄小台灯 + 深蓝夜色
    - 暖黄: #F5D5A0, opacity 0.10, 从下方（台灯感）
    - 深蓝: #2D3A4A, opacity 0.06, 从上方（窗外夜色）
  光型: 两张 radial-gradient 叠加
  小外状态: 眼睛微微眯起（不困，但进入"深夜模式"）
  特殊: 深夜时小外的眼镜边框微光变为暖黄色（平时是蓝灰）
```

### 3.3 技术实现

```css
/* 核心: 小外容器上的伪元素 radial-gradient */
.companion-glow {
  position: relative;
}
.companion-glow::after {
  content: '';
  position: absolute;
  /* 180px 半径，从小外中心向外辐射 */
  width: 360px; height: 360px;
  left: 50%; top: 50%;
  transform: translate(-50%, -50%);
  border-radius: 50%;
  pointer-events: none;
  /* 渐变色由 JS 根据时间动态设置 */
  background: radial-gradient(circle, var(--glow-color) 0%, transparent 70%);
  opacity: var(--glow-opacity);
  transition: opacity 900s linear; /* 15分钟渐变 */
  z-index: -1;
}
```

## 四、科研共情行为系统

### 4.1 设计原则

共情行为不是"小外在表演"，而是"小外在感知"。每个行为的目的是传达：**我注意到你的研究状态了**。

关键约束:
- 共情行为频率极低（每小时不超过 3 次主动共情信号）
- 每个信号持续时间短（不超过 3 秒）
- 信号结束后小外立刻恢复安静姿态
- 不累积、不升级、不形成"互动链"

### 4.2 共情行为清单

#### A. 长时间凝视同一篇文献

```
检测: 页面无滚动、无点击、无键盘操作 > 5 分钟
小外反应: 极小哈欠
  - 嘴微张 2mm（极克制，嘴弧线从直线变成浅弧）
  - 持续 1.5s
  - 完成后轻轻咂嘴（嘴线快速恢复 + 微微弹动）
  - 身体完全不参与（只有嘴部微动）
含义: "你在认真读，我陪着你（但我有一点点困了）"
不是搞笑，是共情——长时间专注是累的，小外知道。
```

#### B. 进度条长时间停滞

```
检测: crawl_jobs status === "running" 且 progress 值 30 秒内变化 < 2%
小外反应: 爪子轻推
  - 右手前爪抬起 8px，向屏幕右侧（进度条方向）轻推一下
  - 像在说 "go on..."
  - 推完后爪子自然回落
  - 持续 1.5s
  - 不重复（同一停滞期只做一次，直到进度再次变化才可能重新触发）
含义: 不是真的在推，而是"我也在等"的陪伴姿态。
```

#### C. 任务完成（搜索/编码/分析）

```
检测: crawl_jobs status → "completed" 或 batch 全部 done
小外反应: 轻敬礼
  - 小外从趴姿变为坐姿（身体抬起 15px）← 这是少数站立姿态之一
  - 右手前爪抬起，轻贴额头侧边（像在说"辛苦了"）
  - 保持 2s
  - 眼睛微眯（满意）
  - 回到趴姿
  - 如果用户深夜完成，额外加眼镜微闪暖金
含义: 不是庆祝，是"你做到了，我看见了"。
这个动作极少出现，因此每次出现都有重量感。
```

#### D. 深夜陪伴

```
检测: 当地时间 23:00-05:00 且页面活跃（有鼠标移动）
小外反应: 陪你熬
  - 眼睛半闭（上下眼睑间距缩小 30%）
  - 身体向用户方向（屏幕内侧）微倾 3px
  - 呼吸频率从 4s/周期 降至 5.5s/周期
  - 不做任何跳跃或大幅度动作
含义: 不是劝你睡觉（那是闹钟该做的），而是"我也在，我们一起"。
```

#### E. 报错后的安静等待

```
检测: API 返回 error 或 crawl_jobs status → "failed"
小外反应: 安静倾听
  - 耳朵向前微倾 5°
  - 头部微歪 3°（比平时困惑歪头更小）
  - 眼睛睁大 10%（关注）
  - 1.5s 后恢复正常
  - 与"报错困惑"(眼镜橙色)同时触发，但动作独立
含义: "出问题了，我在听"
```

### 4.3 共情行为抑制规则

| 抑制条件 | 原因 |
|---|---|
| 用户正在快速操作（30s 内 >10 次点击/键盘） | 在冲刺，不打扰 |
| 用户正在编码页面（coding/[id]） | 编码是高度专注活动 |
| 前一个共情行为结束不足 60s | 不连续刷存在感 |
| 页面不可见 (`document.hidden`) | 用户不在看 |
| 移动端 | 无桌宠 |

## 五、轻量亲密互动

### 5.1 设计原则

亲密互动是**用户发起的、极其缓慢的、微弱的**交互，小外的回应同样微弱。

核心约束:
- 仅响应**极慢速**鼠标移动（<50px/s 经过小外头部区域时）
- 回应幅度极小（3-5px 级别的位移或更小的表情变化）
- 不累积、不形成"好感度"系统
- 快速划过不触发（避免意外互动）

### 5.2 亲密互动清单

#### A. 慢速摸头

```
触发: 鼠标在小外头部 ±30px 区域内，移动速度 <50px/s
小外反应: 微眯眼 + 轻蹭
  - 眼睛眯起（眼睑闭合 40%，比"满意"更深一点）
  - 头部向鼠标移动方向微倾 2-3px（跟随感）
  - 如果鼠标持续在头部区域 >3s，头部会轻微蹭向光标（位移 3px）
  - 鼠标离开后 2s 内恢复正常
视觉信号: 极其微弱。用户可能甚至注意不到小外在回应。
          但潜意识层面会产生"它是有温度的生物"的感觉。
```

#### B. 快速摸头（抑制）

```
触发: 鼠标快速划过（>100px/s）
小外反应: 无。
原因: 快速移动不是亲密信号，可能是用户在工作。
```

#### C. 长时间不互动后的首次接触

```
触发: 用户离开 >30min 后首次移动鼠标经过小外
小外反应: 快速眨眼 × 2（醒来），身体微膨胀 2px（"你回来了"）
持续: 1s
```

### 5.3 不提供的互动

| 互动 | 原因 |
|---|---|
| 点击多次触发特殊动画 | 游戏化 → 不符合学术气质 |
| 拖拽小外 | 小外不是玩具 |
| 鼠标滚轮触发 | 滚轮用于内容滚动，不应被劫持 |
| 右键菜单 | 右键是浏览器功能 |
| 任何形式的"解锁"或"升级" | 小外是陪伴者，不是游戏角色 |

## 七、行为系统总览（v3 完整版）

### 7.1 所有行为清单

```
基础生命 (Layer 1) — 始终运行:
  ├─ 呼吸身体起伏 (4s, 深夜 5.5s)
  ├─ 眨眼 (3-8s 随机, 深夜间隔变长至 5-12s)
  ├─ 耳朵微动 (15-30s 随机)
  ├─ 耳缘毛微飘 (5-10s, 像微风)
  └─ 围巾微飘 (8-15s, 极微弱)

环境感知 (Layer 2) — 被动响应:
  ├─ 眼球跟随鼠标 (rAF, 缓动 300ms)
  ├─ 悬停抬头 (进入 hitbox → 抬头 3px)
  ├─ 点击轻跳 (spring 曲线 0.3s)
  └─ 慢速摸头 (微眯眼 + 轻蹭 3px)

自主行为 (Layer 3) — 低频随机:
  ├─ 歪头 (60-120s 随机)
  ├─ 伸懒腰 (5-10min)
  ├─ 刨爪子 (8-15min, 前爪交替微动)
  ├─ 耳朵单侧轻抖 (20-40s)
  └─ 极小哈欠 (15-30min, 极克制, 仅嘴部微动)

系统联动 (Layer 4) — 事件驱动:
  ├─ 搜索开始 → 围巾飘起 + 眼镜微光 pulse
  ├─ 进度停滞 >30s → 爪子轻推进度条方向
  ├─ 搜索完成 → 轻敬礼 (坐姿 + 爪贴额头)
  ├─ 搜索报错 → 歪头困惑 + 眼镜橙闪 + 耳朵前倾
  ├─ 编码完成 → 微眯眼
  ├─ AI 预读 → 微微前倾
  ├─ 长时间无操作 → 打盹
  └─ 用户回归 → 苏醒伸懒腰

科研共情 (Layer 5) — 低频深度共情:
  ├─ 凝视文献 >5min → 极小哈欠
  ├─ 深夜工作 → 半闭眼 + 身体微倾
  ├─ 报错后 → 安静倾听（耳前倾 + 眼关注）
  └─ 任务完成 → 轻敬礼（极稀有，有重量感）

时间氛围 (Layer 6) — 时段驱动:
  ├─ 晨间: 暖金光 + 抬头欢迎
  ├─ 白天: 无光 (透明融入)
  ├─ 傍晚: 暖琥珀光 + 围巾微松
  └─ 深夜: 双色光 + 半闭眼 + 眼镜暖黄微光
```

### 7.2 行为频率与克制

```
高频 (1-10s):   呼吸、眨眼、眼球跟随、耳毛微飘
```

### 8.2 时间氛围光 Hook

```typescript
function useTimeAtmosphere() {
  const [glow, setGlow] = useState<{ color: string; opacity: number }>({ color: 'transparent', opacity: 0 });

  useEffect(() => {
    function update() {
      const hour = new Date().getHours();
      if (hour >= 6 && hour < 10)       setGlow({ color: '#F5E6C8', opacity: 0.12 }); // 晨间
      else if (hour >= 10 && hour < 17)  setGlow({ color: 'transparent', opacity: 0 }); // 白天
      else if (hour >= 17 && hour < 20)  setGlow({ color: '#F0D5B0', opacity: 0.10 }); // 傍晚
      else                               setGlow({ color: '#F5D5A0', opacity: 0.10 }); // 深夜(暖) + 深蓝由 CSS 叠加
    }
    update();
    const timer = setInterval(update, 600_000); // 每10分钟检查一次
    return () => clearInterval(timer);
  }, []);

  return glow;
}
```

### 8.3 鼠标速度检测 Hook

```typescript
function useMouseSpeed(): number {
  const speedRef = useRef(0);
  const lastPos = useRef({ x: 0, y: 0, t: 0 });

  useEffect(() => {
    const handler = (e: MouseEvent) => {
      const now = performance.now();
      const dx = e.clientX - lastPos.current.x;
      const dy = e.clientY - lastPos.current.y;
      const dt = now - lastPos.current.t;
      speedRef.current = dt > 0 ? Math.hypot(dx, dy) / (dt / 1000) : 0;
      lastPos.current = { x: e.clientX, y: e.clientY, t: now };
    };
    window.addEventListener('mousemove', handler, { passive: true });
    return () => window.removeEventListener('mousemove', handler);
  }, []);

  return speedRef.current; // px/s
}
```

## 九、设计红线和禁止清单

### 9.1 绝对禁止

| 行为 | 原因 |
|---|---|
| 弹窗、气泡、文字提示 | 打扰学术工作流 |
| 高频动作（>3次/分钟） | 分散注意力 |
| 卖萌表情（星星眼、脸红、爱心） | 二次元化 |
| 声音效果（v1） | 学术环境需要安静 |
| 积分解锁、签到、打卡 | 游戏化 |
| 拟人化语言或"说话" | 小外是犬，不是人 |
| 强制互动（如不点它会一直动） | 情感绑架 |
| 遮挡内容区域 | 功能性底线 |

### 9.2 谨慎边界

| 行为 | 允许但需克制 |
|---|---|
| 轻敬礼 | 仅在搜索完成时，每天最多 3 次 |
| 慢速摸头回应 | 仅极慢鼠标移动触发 |
| 氛围光 | 仅 180px 范围，不影响页面 |
| 深夜陪伴 | 仅半闭眼和微倾，不做任何"催睡觉"动作 |
| 节日皮肤 | 仅围巾换色，不改变角色本体 |
| 打哈欠 | 极小嘴部动作，不张大嘴，不伸舌头 |

### 10.2 研究日记（v2.0）

每周一，小外在清晨首次打开时，围巾侧边出现一个极小的数字：
- "7" = 上周完成 7 篇编码
- 数字在 5s 内淡出
- 不保留、不累积、不形成"排行榜"

### 10.3 多小外（v3.0，远期）

同一研究团队的不同用户，小外围巾颜色不同。用户 A 的小外是蓝色围巾，用户 B 是灰粉色。当同一篇文章被多人编码时，两人的小外在各自屏幕上有相同的"点头"时刻——不通讯、不联机，只是各自独立但同步。

## 附录 B: 核心设计决策

| 决策 | 原因 |
|---|---|
| 线条抖动而非光滑矢量 | 光滑=工业感，微抖=手账感/温度感 |
| 耳缘毛但不过多 | 飘逸毛传达"柔软"，过多变"毛绒玩具" |
| 时间氛围光而非全局暗色模式 | 氛围光是陪伴信号，全局暗色是功能信号 |
| 轻敬礼而非鼓掌 | 敬礼=尊重+认可，鼓掌=庆祝（偏娱乐） |
| 爪子推进度条 | 非功能性动作，纯粹的共情表达 |
| 极慢鼠标摸头触发 | 只有"刻意温柔"才触发回应，快速操作不触发 |
| 莫兰迪色系 | 低饱和=耐看=学术环境不刺眼 |
| 眼镜起雾 | 极小细节，增加生物温暖感，不影响功能 |
