第三章 · 内容提取

批量文本清洗

前端驱动轮询架构

1. 点击"清洗已有全文"按钮
→
2. GET /api/articles/batch-clean?limit=50
→
3. 每 2 秒轮询下一批，直到 done=true
→
4. 安全检查：字数下降 > 80% → 标记 needs_review

幂等性设计

metadata.cleaned_at 时间戳

每篇处理过的文章（包括跳过的）都写入清洗时间戳，查询时排除已有时间戳的文章，确保不重复清洗。

安全机制

字数下降超 80% 且原文 > 100 词 → 不覆盖原文，标记 metadata.needs_review = true，记录原始和清洗后字数。

第四章 · AI 预读

AI-人工协作编码模式

AI 的角色：辅助者，非替代者

  • 效率工具：自动提取摘要、术语、信源统计
  • 参考锚点：情感/框架/叙事的初步判断
  • 一致性锚点：AI 分析可作为编码校验基准
  • 规模化：800+ 篇文章的批量预处理

人工的不可替代性

  • 决策者：最终编码决策由人工做出
  • 质量控制：双人编码 + Kappa 检验
  • 理论建构：基于编码数据进行话语分析
  • 文化理解：跨文化语境的深度解读

计算公式

agreement.ts
// κ = (Po - Pe) / (1 - Pe) Po = 观察一致率 // 两人实际选择相同节点的比例 Pe = 期望一致率 // 若随机编码，预期一致的比例 // Po = Σ min(pA_i, pB_i) // Pe = Σ pA_i × pB_i // κ = (Po - Pe) / (1 - Pe)

示例

定量：NYT 的"威胁"框架占比 45%，显著高于 Guardian 的 20%
定性：NYT 的威胁框架多采用 conflict-driven 叙事，信源以西方专家为主
综合：NYT 通过选择特定信源和叙事策略来建构中国式现代化的"威胁"形象
